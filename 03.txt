3. 작업 수행 방식
- 배치/타임셰어링(시분할)/분산처리/실시간

배치: 
모든 시스템을 중앙(전자계산소 등)에서 관리 및 운영
사용자의 요청 작업(천공카드 등)을 일정 시간 모아두었다가 한 번에 처리

타임셰어링:
여러 사용자가 자원을 동시에 사용. OS가 파일 시스템 및 가상 메모리 관리
사용자 지향적. 대화형 시스템, 단말기 사용.

PC:
개인이 시스템 전체 독점
CPU활용률은 고려의 대상이 아니게 바뀜
OS가 상대적으로 단순함. 하지만 다양한 사용자 지원 기능 지원
빠르지만 성능이 낮음

Parrallel Processing System
단일 시스템 내에서 둘 이상의 프로세서 사용
동시에 둘 이상의 프로세스 지원
프로세서간 관계 및 역할 관리 필요하다.

분산처리
네트워크를 기반으로 구축된 병렬 처리 시스템

리얼타임
작업처리에 제한시간(데드라인)을 갖는 시스템
제한 시간 내에 서비스를 제공하는 것이 자원 활용 효율보다 중요
태스크의 종류: hard real-time task : 시간 제약을 지키지 못하는 경우 시스템에 치명적
		  	         예를 들어 발전소, 무기, 핵
	       soft real-time task : 동영상 재생 화질이 박살나면 화남
	       non real-time task
